## 算法思想

**贪心法**

## 思路分析

这一题和**435.无重叠区间**的处理模式基本一致，这里仅指出不同点：

1. 此处重叠的判据包含了仅端点值相同，e.g.认为[2,3][3,4]是重叠的；

2. 此处的count的计算逻辑不同，在此前的那道题中，count记录的是需要删除的重复区间数，但是此处询问的是需要射出的箭的数量，由于箭是沿直线无限射出的，因此重叠与不重叠的区别在于：

重叠的区间一箭就可以了，其余的则额外需要一箭。即：默认情况下，初始的count应该设置为所有待检测元素的个数，而当检测到区间重合时，每检测到一个则说明此处少射一箭，count--即可。

## AC代码如下:

```C++
#include <bits/stdc++.h>
using namespace std;
class Solution {
public:
    int findMinArrowShots(vector<vector<int>>& points) {
    sort(points.begin(),points.end(),[](const vector<int>&a, const vector<int>&b)
    {
        return a[1]<b[1];
    });

    int count = points.size();
    int prev = points[0][1];
    for(int i=1;i<points.size();i++)
    {
        if(points[i][0]<=prev)
        {
            count--;
        }
        else
        {
            prev = points[i][1];
        }
    }
    return count;
    }
};

```
