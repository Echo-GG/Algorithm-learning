## 算法思想

**贪心法**

## 思路分析：

容易观察到:当 n = 1 或 n = 2 时，无论如何数列是可以在1步内被调整为非递减形式的。

接下来只需要检查数列中的非递减的子区间个数，当子区间个数 > 1 时，毫无疑问原数列无法在1步内被更改为非递减数列。

当子区间个数 = 1 时，此时我们记录前一子区间的结束时元素的下标idx ，只有当 nums[idx+2] > nums[idx] && nums[idx-1] > nums[idx+1]时，无法在1步内将原数列调整为非递减数列。

特别地，n = 3 时一定可以在此时非递减子区间个数为1的情况下，于1步内完成更改。

## AC代码如下:
```C++
#include <bits/stdc++.h>
using namespace std;
class Solution {
public:
    bool checkPossibility(vector<int>& nums) {
        int n = nums.size();
        if(n==1 || n==2)
            return true;
        int cnt = 0;
        int idx = -1;
        for(int i=0;i<n-1;i++)
        {
            if(nums[i+1]-nums[i]<0)
            {
                cnt++;
                if(cnt>1)
                    return false;
                idx = i;
            }
        }
        if(idx>0 && idx< n-2)
        {
            if(nums[idx-1] >nums[idx+1] && nums[idx] > nums[idx+2])
                return false;
        }
        return true;
    }
};
```
