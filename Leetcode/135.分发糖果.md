## 算法思想

**贪心法**

## 思路说明

贪心策略不一定需要排序或者选择，本题采取两次遍历的方法实现每一次的局部最优：

初始化每个孩子对应的初始糖果数均为1：

1. 从左往右遍历，当右手边孩子的分数高于与之相邻的左手边的孩子时，右手边孩子的糖果数为左手边孩子的糖果数+1；
   
2. 从右往左遍历，当左手边孩子的分数高于与之相邻的右手边的孩子时：

  此时左手边孩子的糖果数存在两种情况：
  
  (1). 为右手边孩子的糖果数+1；
  
  (2). 当此时右手边孩子的糖果数+1不大于原来第一次遍历时在此处填入的孩子应分配的糖果数，则此处仍保留第一次遍历时填入的糖果数。

**i.e. 在每次遍历中，只考虑并更新相邻一侧的大小关系。**


## 以下为AC代码:

```C++
#include<bits/stdc++.h>
using namespace std;
class Solution {
public:
    int candy(vector<int>& ratings) {
        int size = ratings.size();
        vector<int> candies(size,1);
        if (size<2)
            return size;
        for(int i=0;i<size-1;i++)
        {
            if(ratings[i]<ratings[i+1])
            {
                candies[i+1] = candies[i] + 1;
            }
        }
        for(int i=size-1;i>0;i--)
        {
            if(ratings[i]<ratings[i-1])
            {
                candies[i-1] = max(candies[i-1],candies[i]+1);
            }
        }

        return accumulate(candies.begin(),candies.end(),0);
    }
};
```
