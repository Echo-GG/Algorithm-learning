## 算法思想

使用快慢指针。

## 过程描述

1. 如果向量长度为0则直接返回0

2. 对于非空向量定义一对快慢指针,fast与slow初始时均对应数组下标1

    初始时同时指向1而不是0主要出于两个考虑：
        
        (1). 如果向量中仅含有一个元素，那么显然返回结果为1，故不具备讨论意义，天然的不会出现重复情况。
        
        (2). 考虑到函数返回的是非重复的元素的个数，这里考虑用slow的值来作为返回值，容易考虑元素为1和2的情形，显然对应的slow的值恰为正确的返回值。
        
    另一种理解方式是：
    slow初始值为1，即代表下标为0的那个元素，此后slow的每一次增加，都是下标为0的元素之后的非重复的元素的个数增加，故slow的值为函数的正确返回值。

3.当fast指针还没有越界时(遍历整个向量)：

    如果相邻的一前一后的两个元素为相等的元素，那么我们采取继续移动fast指针的方式将其略过,slow指针此时仍指在原地；
    
    反之如果不等，则将下标为slow的指在原地位置处指针中的值，设置为新的不同于原重复元素的下标为此时fast的元素。
    
    这样就达到了“删除”旧的多余重复元素的效果（即覆盖掉原slow指针指向的位于原地位置处的指针中的值）。
    
    然后同时移动fast和slow指针，考虑下一组的情形。
    
    由于无论如何都要移动fast指针，所以考虑将fast++置于条件判断语句之外。

4. return slow;

    那么此时slow前的数组即为正确的不重复的子数组序列，slow后(包括此时的slow)的（与fast相间隔的）元素，即为与slow前(不包括此时的slow)出现过的元素相重复的相同元素（尚未覆盖完但fast已越界）。

## 以下为AC代码:

```C++
#include <bits/stdc++.h>
using namespace std;
class Solution {
public:
    int removeDuplicates(vector<int>& nums) {
        int n = nums.size();
        if (n == 0) 
        {
            return 0;
        }

        int fast = 1, slow = 1;
        while (fast < n) 
        {
            if (nums[fast] != nums[fast - 1]) 
            {
                nums[slow] = nums[fast];
                ++slow;
            }

            ++fast;
        }

        return slow;
    }
};
```
